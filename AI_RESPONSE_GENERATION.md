# Updated Response Generation with AI - Usage Guide

## What Changed

The response generation system now has two modes:

### 1. **AI-Powered Mode** (Recommended)

Uses AI to intelligently extract response schemas from controller code:

```bash
python main.py generate-docs --backend ./laravel-project --provider ollama --only-backend
```

- ✅ Analyzes controller code to extract response details
- ✅ Determines response type (json, array, paginated, etc.)
- ✅ Lists all response fields with inferred types
- ✅ Adds descriptions for responses
- ✅ Works even if static analysis can't detect responses

### 2. **Static Mode** (Fallback)

When AI is disabled or unavailable:

```bash
python main.py generate-docs --backend ./laravel-project --provider ollama --no-ai --only-backend
```

- ✅ Uses regex patterns to detect response fields
- ✅ Shows `Not detected in static analysis...` for complex responses
- ✅ Fast, no API calls needed

## How AI Response Extraction Works

1. **Gets controller code** for the action:
   - Extracts PHP function body
   - Looks for return statements
   - Analyzes response()->json() calls

2. **Sends to AI with context**:
   - Full code snippet
   - HTTP method
   - Route path
   - Controller name
   - Action name

3. **AI analyzes and returns**:
   - Response type (json, array, paginated, etc.)
   - All response fields with types
   - Description of what's returned

4. **Formatted as markdown**:
   - Clean JSON structure
   - Field types inferred
   - Human-readable description

## Troubleshooting

**Q: Getting "Not detected in static analysis"?**
A: That's the fallback message. Make sure:

1. Ollama is running: `ollama list`
2. Not using `--no-ai` flag
3. Controller code has return statements

**Q: Ollama not responding?**
A:

```bash
# Check if Ollama is running
ollama list

# If not, start it
ollama serve

# Verify model available
ollama list | grep qwen
```

**Q: Response types showing as generic `<value>`?**
A: AI couldn't infer the type. This happens when:

- Code is too complex
- Model isn't powerful enough
- Dynamic responses not static

**Q: Want faster generation?**
A: Use smaller model:

```bash
python main.py generate-docs \
  --backend ./laravel \
  --provider ollama \
  --model phi3.5:3.8b \  # Smaller, faster
  --only-backend
```
